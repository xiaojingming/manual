# 交互模式

> CSC 会话中键盘快捷键、输入模式和交互功能的完整参考。

## 键盘快捷键

> **注意：** 键盘快捷键可能因平台和终端而异。按 `?` 查看当前环境可用的快捷键。

**macOS 用户**：Option/Alt 键快捷键（`Alt+B`、`Alt+F`、`Alt+Y`、`Alt+M`、`Alt+P`、`Alt+T`）需要在终端中将 Option 配置为 Meta：

* **iTerm2**：设置 → 配置文件 → 按键 → 将左/右 Option 键设置为 "Esc+"
* **Terminal.app**：设置 → 配置文件 → 键盘 → 勾选 "Use Option as Meta Key"
* **VS Code**：在 VS Code 设置中设置 `"terminal.integrated.macOptionIsMeta": true`

有关详细信息，请参阅终端配置。

## 命令历史

CSC 为当前会话维护命令历史：

* 输入历史按工作目录存储
* 运行 `/clear` 开始新会话时，输入历史会重置。之前的会话对话会被保留，可以恢复。
* 使用上/下箭头导航（参见上方的键盘快捷键）
* **注意**：历史展开（`!`）默认被禁用

### 使用 Ctrl+R 反向搜索

按 `Ctrl+R` 交互式搜索命令历史：

1. **开始搜索**：按 `Ctrl+R` 激活反向历史搜索
2. **输入查询**：输入要在之前命令中搜索的文本。搜索词在匹配结果中高亮显示
3. **浏览匹配**：再次按 `Ctrl+R` 循环浏览更早的匹配
4. **接受匹配**：
   * 按 `Tab` 或 `Esc` 接受当前匹配并继续编辑
   * 按 `Enter` 接受并立即执行命令
5. **取消搜索**：
   * 按 `Ctrl+C` 取消并恢复原始输入
   * 在空搜索时按 `Backspace` 取消

搜索会显示匹配的命令并高亮搜索词，方便你查找和复用之前的输入。

## 后台 bash 命令

CSC 支持在后台运行 bash 命令，允许你在长时间运行的进程执行时继续工作。

### 后台运行的工作方式

当 CSC 在后台运行命令时，它会异步执行命令并立即返回一个后台任务 ID。CSC 可以在命令继续在后台执行的同时响应新的提示。

要在后台运行命令，你可以：

* 提示 CSC 在后台运行命令
* 按 Ctrl+B 将常规 Bash 工具调用移至后台。（Tmux 用户由于 tmux 的前缀键，必须按两次 Ctrl+B。）

**主要特性：**

* 输出写入文件，CSC 可以使用 Read 工具检索
* 后台任务具有唯一 ID，用于跟踪和输出检索
* CSC 退出时后台任务会自动清理
* 如果输出超过 5GB，后台任务会自动终止，并在 stderr 中说明原因

要禁用所有后台任务功能，将 `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS` 环境变量设置为 `1`。有关详细信息，请参阅环境变量。

**常见的后台命令：**

* 构建工具（webpack、vite、make）
* 包管理器（npm、yarn、pnpm）
* 测试运行器（jest、pytest）
* 开发服务器
* 长时间运行的进程（docker、terraform）

### 使用 `!` 前缀的 Bash 模式

通过在输入前添加 `!` 前缀，直接运行 bash 命令而无需经过 CSC：

```bash
! npm test
! git status
! ls -la
```

Bash 模式：

* 将命令及其输出添加到对话上下文
* 显示实时进度和输出
* 支持相同的 `Ctrl+B` 后台运行，适用于长时间运行的命令
* 不需要 CSC 解释或批准命令
* 支持基于历史的自动补全：输入部分命令并按 **Tab** 从当前项目中之前的 `!` 命令中补全
* 在空提示下按 `Escape`、`Backspace` 或 `Ctrl+U` 退出
* 将以 `!` 开头的文本粘贴到空提示中会自动进入 bash 模式，与手动输入 `!` 的行为一致

这对于在进行对话的同时进行快速 shell 操作非常有用。

## 提示建议

首次打开会话时，提示输入中会出现一个灰色的示例命令，帮助你入门。CSC 从项目的 git 历史中选择此建议，因此它反映了你最近一直在处理的文件。

在 CSC 回复后，建议会根据你的对话历史继续出现，例如多部分请求的后续步骤或工作流的自然延续。

* 按 **Tab** 或 **右箭头** 接受建议，或按 **Enter** 接受并提交
* 开始输入以关闭建议

建议作为后台请求运行，复用父对话的提示缓存，因此额外成本极低。当缓存冷启动时，CSC 会跳过建议生成以避免不必要的成本。

在对话的第一轮之后、非交互模式和计划模式中，建议会自动跳过。

要完全禁用提示建议，设置环境变量或在 `/config` 中切换设置：

```bash
export CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false
```

## 使用 /btw 进行附带提问

使用 `/btw` 对当前工作进行快速提问，而不会添加到对话历史中。当你想要快速获取答案但不想使主要上下文混乱或使 CSC 偏离长时间运行的任务时，这非常有用。

```
/btw 那个配置文件叫什么名字来着？
```

附带提问对当前对话有完全可见性，因此你可以询问 CSC 已经读取的代码、之前做出的决定或会话中的任何其他内容。问题和答案是临时的：它们出现在可关闭的覆盖层中，永远不会进入对话历史。

* **在 CSC 工作时可用**：你甚至可以在 CSC 处理回复时运行 `/btw`。附带提问独立运行，不会中断主轮次。
* **无工具访问**：附带提问仅从上下文中已有的内容回答。CSC 在回答附带提问时无法读取文件、运行命令或搜索。
* **单次回复**：没有后续轮次。如果需要来回对话，请使用普通提示。
* **低成本**：附带提问复用父对话的提示缓存，因此额外成本极低。

按 **Space**、**Enter** 或 **Escape** 关闭答案并返回提示。

`/btw` 是子代理的反面：它可以看到你的完整对话但没有工具，而子代理拥有完整的工具但从空上下文开始。使用 `/btw` 询问 CSC 在本次会话中已经知道的内容；使用子代理去查找新信息。

## 任务列表

在处理复杂的多步骤工作时，CSC 会创建任务列表来跟踪进度。任务显示在终端的状态区域，带有指示待处理、进行中或已完成的标记。

* 按 `Ctrl+T` 切换任务列表视图。显示一次最多显示 10 个任务
* 要查看所有任务或清除它们，直接询问 CSC："显示所有任务" 或 "清除所有任务"
* 任务在上下文压缩后仍然保留，帮助 CSC 在较大的项目中保持组织
* 要跨会话共享任务列表，设置 `CLAUDE_CODE_TASK_LIST_ID` 以使用 `~/.claude/tasks/` 中的命名目录：`CLAUDE_CODE_TASK_LIST_ID=my-project csc`

## PR 审查状态

当在具有开放拉取请求的分支上工作时，CSC 在页脚显示一个可点击的 PR 链接（例如 "PR #446"）。链接带有彩色下划线，指示审查状态：

* 绿色：已批准
* 黄色：待审查
* 红色：需要修改
* 灰色：草稿
* 紫色：已合并

`Cmd+点击`（Mac）或 `Ctrl+点击`（Windows/Linux）链接可在浏览器中打开拉取请求。状态每 60 秒自动更新。

> **注意：** PR 状态需要安装并认证 `gh` CLI（`gh auth login`）。
